⚡ Hugo 的工作过程：
.md + 模板 = 网页

Lesson 3 · 预计阅读：15 分钟 · 难度：入门 · 关联：术语表

1. Hugo 的核心公式

Hugo 做的事可以浓缩成一个公式：

.md 文件（内容） + Go 模板（布局） = 完整的 HTML 网页 │ │ │ content/ 目录 layouts/ 目录 public/ 目录 （文章、页面） （骨架 + 样式） （最终网站）

这就是静态站点生成器的本质。它不是在用户访问时才拼装页面——而是在"构建"时就把所有页面提前生成好。用户访问时直接拿到现成的 HTML，所以速度飞快。

🔑 一句话记住 Hugo

内容（.md） + 模板（HTML） → 网站（public/）

2. .md 文件里有什么

打开 blog/content/articles/ 目录下任何一个 .md 文件，你会看到两部分：

---
title: "[EN] OpenAI 预览 GPT-5.6 Sol..."
date: "2026-07-04T06:39:03+08:00"
categories:
  - AI 技术
tags:
  - OpenAI
  - GPT-5.6 Sol
summary: "OpenAI 发布了下一代模型..."
aigc: true
---
← 这是 frontmatter（元数据）

OpenAI 发布了下一代模型 GPT-5.6 Sol 的预览...
← 这是正文（Markdown）
部分叫什么作用
--- 之间的内容Frontmatter文章的元数据（标题、日期、分类等）。模板用 .Title、.Date、.Params.summary 来读取这些字段。
--- 下面的内容正文（Body）文章的实际内容，Markdown 格式。模板里用 .Content 输出。
💡 Frontmatter 的真相

Frontmatter 就是一篇"写给 Hugo 看的文章简介"。它告诉 Hugo：这篇文章的标题是什么、什么时候发布的、属于什么分类。模板根据这些信息来决定怎么展示。Template 中 .Title 读的就是 frontmatter 里的 title 字段，.Params.summary 读的是 summary 字段——.Params 是"任意自定义字段"的统称。

3. 模板里有什么

打开 blog/layouts/_default/single.html：

{{ define "main" }}
<article>
  <h1>{{ .Title }}</h1>              ← 输出文章标题
  <time>{{ .Date.Format "2006-01-02" }}</time>  ← 输出日期
  {{ if .Params.aigc }}
    <span>AI 辅助生成</span>         ← 只有 aigc=true 才显示
  {{ end }}
  <div>{{ .Content }}</div>         ← 输出正文（Markdown 转 HTML）
{{ end }}

这里 {{ .Title }}、{{ .Content }} 这些就是 Go Template 标签——它们是 Hugo 的"占位符"。

模板标签读取的位置示例值
{{ .Title }}当前页面的 frontmatter title"[EN] OpenAI 预览 GPT-5.6 Sol..."
{{ .Content }}当前页面的 Markdown 正文（已转 HTML）<p>OpenAI 发布了...</p>
{{ .Date }}frontmatter 中的 date2026-07-04
{{ .Params.summary }}frontmatter 中的自定义字段 summary"OpenAI 发布了..."
{{ .Site.Title }}站点配置（hugo.toml）中的 title"AI 早报"

4. 合并过程：Hugo 怎么决定用哪个模板？

Hugo 有一套模板查找规则（Lookup Order）。对于一篇文章（叫 single page），它会按这个顺序找模板：

layouts/ ← Hugo 先看 layouts/ 目录 ├── _default/ │ ├── baseof.html ← 这是"骨架" │ └── single.html ← 这是"填充物" ├── index.html ← 首页模板 └── 404.html ← 404 页面

查找逻辑：

  1. Hugo 读取 .md 文件，确定页面类型（这是"单页" single page）
  2. Hugo 去 layouts/ 里找 single.html
  3. 找到 layouts/_default/single.html，用它来渲染内容
  4. baseof.html 会被自动作为"外层骨架"包裹进来

baseof.html 和 single.html 怎么配合？

这是 Hugo 模板系统中最重要的关系：

baseof.html（骨架） single.html（填充） ┌──────────────────────┐ ┌──────────────────────┐ │ <html> │ │ {{ define "main" }} │ │ <head>...</head> │ │ <h1>.Title</h1> │ │ <body> │ │ .Content │ │ {{ block "main" . }} │ ← ─ ─ │ ... │ │ {{ end }} │ │ {{ end }} │ │ </body> │ └──────────────────────┘ │ </html> │ └──────────────────────┘

baseof.html 里定义了一个"插槽"：{{ block "main" . }}...{{ end }}。

single.html 里用 {{ define "main" }}...{{ end }} 填充这个插槽。

最终结果：Hugo 把 single.html 的内容"塞进" baseof.html 的插槽位置，合成一个完整的页面。

🎯 类比：就像"相框 + 照片"

baseof.html 是相框——每个页面都一样的外框（header、footer、css）。single.html 是照片——每篇文章不同的内容。Hugo 的工作就是选择合适的照片放进相框里。

5. 两种页面类型：List vs Single

Hugo 把页面分成两大类：

类型模板做什么用.Pages
List（列表页） list.html 显示一组文章的概览（首页、分类页、标签页） 有（多篇文章）
Single（单页） single.html 显示一篇文章的完整内容 没有（只有自身）

首页是一个特殊的列表页，它的模板是 layouts/index.html。在我们的项目中，首页按日期分组显示所有文章。

6. 到底怎么合并——以这篇文章为例

当用户访问 /articles/en-openai-gpt-56-sol/ 时，Hugo 的构建过程：

┌─────────────────────────────────────────────────────┐ │ Step 1：读取 .md 文件 │ │ /content/articles/2026-07-04-en-openai-gpt-56-sol.md │ │ 提取 frontmatter + Markdown 正文 │ └──────────────────────┬──────────────────────────────┘ ▼ ┌─────────────────────────────────────────────────────┐ │ Step 2：选择模板 │ │ 类型 = single → 找 _default/single.html │ │ 有 baseof.html → 作为骨架 │ └──────────────────────┬──────────────────────────────┘ ▼ ┌─────────────────────────────────────────────────────┐ │ Step 3：合并渲染 │ │ │ │ .Title → "[EN] OpenAI 预览 GPT-5.6 Sol..." │ │ .Content → Markdown → HTML（标题、段落、列表） │ │ .Date → "2026-07-04T06:39:03+08:00" │ │ .Params.aigc → true → 显示"AI 辅助生成"标签 │ │ .Params.source → "OpenAI Blog" │ │ │ └──────────────────────┬──────────────────────────────┘ ▼ ┌─────────────────────────────────────────────────────┐ │ Step 4：输出到 /public/articles/en-openai-gpt-56-sol/ │ │ index.html — 完整的 HTML 页面 │ └─────────────────────────────────────────────────────┘

实际生成的 HTML 长什么样

{{ .Title }} 变成：

<h1 class="text-3xl font-bold text-gray-900 dark:text-white leading-tight">
  [EN] OpenAI 预览 GPT-5.6 Sol：新一代模型强化编程、科学与网络安全能力
</h1>

{{ if .Params.aigc }} 变成：

<span class="tag">AI 辅助生成</span>

{{ .Content }} 变成：

<h2>关键要点</h2>
<ul>
  <li><strong>GPT-5.6 Sol 在编程、科学和网络安全三大关键领域能力显著提升。</strong></li>
  ...
</ul>

最终，所有这些 HTML 片段被组合到 baseof.html 的骨架里，成为一个完整的页面。

7. .md 文件在构建时才读取

这一点很重要：

  • 你在 content/articles/ 里放一个新的 .md 文件，Hugo 并不会立刻知道
  • 你必须重新构建（执行 hugo 命令），Hugo 才会读取它
  • 构建 → 输出新的 public/ → 部署上线

在我们的系统中，Pipeline 生成 .md 后自动 commit → 触发 Cloudflare Pages 重新构建和部署。所以最终效果就像"实时更新"一样。
