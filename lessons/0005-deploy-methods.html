1. 两种思路

把博客放到公网上，本质上只有两条路：

方式怎么做典型服务
静态托管 把生成好的 HTML 文件上传到 CDN Cloudflare Pages、Vercel、Netlify、GitHub Pages
服务器托管 租一台云服务器，自己装 Web 服务软件 阿里云、腾讯云、AWS EC2、VPS

静态托管适合 Hugo、Astro 这类静态站点生成器。你只需要上传最终文件，托管服务负责分发。服务器托管适合 WordPress、动态网站——需要运行程序、连接数据库的场景。

我们这个项目用的是静态托管，所以下面重点看这条路。

💡 静态托管为什么便宜

因为 CDN 只管"把已经生成好的文件发出去"，不需要在服务器上运行任何程序。这就像印刷厂印好报纸后让快递员派送——派送本身很便宜。而服务器托管像是"每家每户临时打印当天的报纸"——贵得多。

3. 两种部署方式

不管你选哪个平台，部署都有两种搞法：

方式 A：平台自动构建（Git 集成）

你授权平台访问你的 GitHub 仓库，平台会自动：

  1. 检测到新 push
  2. 拉代码到自己服务器上
  3. 执行你配置的构建命令（如 hugo --gc --minify）
  4. 把输出目录部署上线

你不需要自己准备构建环境。这就是 Cloudflare Pages 的"Connect to Git"模式。但你的项目在 monorepo 的子目录里（blog/），Cloudflare 的新界面把这个配置入口藏起来了，导致之前没成功。

方式 B：外部 CI 推送到平台

你在 GitHub Actions 里自己构建好，然后用 CLI 工具把产物推给托管平台：

GitHub Actions Cloudflare Pages ┌──────────────┐ ┌──────────────────┐ │ 安装 Hugo │ │ │ │ hugo 构建 │ │ 收到产物 → 部署 │ │ │────► │ │ │ wrangler │ │ CDN 分发 │ │ pages deploy│ │ │ └──────────────┘ └──────────────────┘

这就是我们项目用的方式。优点是你完全控制构建过程，不依赖平台的构建环境。

4. 为什么本项目最终选了 Cloudflare Pages + GitHub Actions

回顾一下我们的选择过程：

  1. GitHub Pages — 最简单，但 monorepo 子目录配置复杂
  2. Vercel — Hugo 构建需要额外配置
  3. Cloudflare Pages — 无限带宽 + 最佳免费额度 + Wrangler CLI 成熟

最终的决定是：Cloudflare Pages（做存储和分发）+ GitHub Actions（做构建）。理由：

  • GitHub Actions 的 2000 分钟/月免费额度比 Cloudflare Pages 自带的 500 次构建更宽裕
  • Pipeline 和部署在同一个 CI 平台上，统一管理
  • Cloudflare 的无限带宽是最大优势——个人博客流量增长时不用担心账单

6. 自定义域名

所有平台都支持绑定自己的域名。流程是：

  1. 在域名注册商（如 Namesilo、GoDaddy、阿里云）买一个域名（如 ainews.com）
  2. 在托管平台添加域名，获得 DNS 记录（通常是 CNAME 或 A 记录）
  3. 去域名注册商的管理面板添加这些 DNS 记录
  4. 等待 DNS 生效（几分钟到 48 小时）
  5. 自动获得免费的 HTTPS 证书

如果域名也托管在 Cloudflare，绑定过程可以一步完成——Cloudflare 会自动检测到。所以常见组合是"在 Namesilo 买域名，DNS 托管到 Cloudflare（免费），然后绑定到 Cloudflare Pages"。

7. 服务器托管（补充）

前面的内容主要聚焦静态托管。这一节专门讲另一条路——服务器托管。

7.1 什么是服务器托管

服务器托管就是你租一台远程电脑，在上面装软件、跑程序。这台电脑 24 小时开机，有公网 IP，全世界都能访问。

类型白话解释典型价格
VPS（虚拟专用服务器） 一台大服务器上划分出多个"小房间"，你租一间 $3-10/月
云服务器 弹性伸缩的 VPS，可以随时升级配置 ¥50-200/月
独立服务器 整台机器全归你用 $50-200/月

对于个人博客，VPS 是最常见的起步选择。

7.5 什么情况下需要服务器托管

为什么我们的项目不用服务器？因为不需要。但如果出现以下需求，就该考虑上服务器了：

  • 需要用户登录 — 比如读者注册、评论系统
  • 需要数据库 — 比如存储用户数据、文章阅读量
  • 需要后台运行程序 — 比如自定义的 API、Webhook
  • 需要 PHP/Node/Python 运行时 — 比如 WordPress、Ghost
  • 国内需要备案 — 如果目标读者主要在国内，用国内服务器速度更快

7.6 "静态托管 + 轻量后端"混合方案

一个常见的折中方案：博客本身用静态托管（免费），评论区等动态功能用一个轻量后端服务。比如：

  • 评论区 — 用 Giscus（基于 GitHub Discussions）或 Waline（基于 LeanCloud）
  • 阅读量统计 — 用 Plausible（自托管）或 Umami（免费分析）
  • 搜索功能 — 用 Pagefind（构建时生成搜索索引）
  • 表单/订阅 — 用 Formspree 或 Getform

这些服务大部分都有免费额度，不需要自己租服务器。这样你可以既享受静态托管的低成本，又有部分动态功能。

7.7 手动部署到 VPS 的简单步骤

如果你真的想试一把，步骤大概这样：

# 1. 在服务器上安装 Nginx
sudo apt update
sudo apt install nginx

# 2. 配置 Nginx 站点
sudo nano /etc/nginx/sites-available/blog.example.com
# → 设置 root 指向你的静态文件目录

# 3. 启动
sudo ln -s /etc/nginx/sites-available/blog.example.com /etc/nginx/sites-enabled/
sudo systemctl reload nginx

# 4. 配置 HTTPS（免费）
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d blog.example.com

# 5. 部署新版本：用 scp 或 rsync 把本地构建好的文件传到服务器
rsync -avz public/ user@yourserver:/var/www/blog/

当然，这一步可以和 GitHub Actions 结合——在 Actions 里构建好后，通过 SSH 自动上传到服务器。

8. 内网穿透：临时测试方案

如果你的博客只在本地跑（hugo server），只有你电脑自己能访问。但有时候你想让别人临时看看效果——比如给朋友验证、用手机测试、或者给甲方演示——又不想正式部署到公网。

这时候就需要内网穿透：把你本机的端口（如 localhost:1313）通过一个中间服务器暴露到公网上。

8.1 原理

你的电脑 穿透服务 访问者 localhost:1313 服务器 浏览器 ┌──────────────┐ ┌──────────────┐ ┌──────────┐ │ Hugo Server │───────►│ 中转服务器 │◄───────│ 朋友手机 │ │ 127.0.0.1 │ │ 临时 URL │ │ 随便哪里 │ └──────────────┘ └──────────────┘ └──────────┘

你只需要在终端跑一行命令，穿透服务会给你一个公网 URL（如 https://abc123.ngrok.io），任何人打开这个 URL 就能看到你本机的网页。

你的博客一直都在你自己电脑上，穿透服务只负责"转发"流量，不存储内容。

8.2 常见的穿透工具

工具免费额度命令示例特点
ngrok 1 个隧道，40 连接/分钟 ngrok http 1313 最知名，速度稳定，有 Web 调试界面
frp 需自建服务器 ./frpc -c frpc.toml 开源，功能强，可自建中转服务
Tailscale Funnel 免费（3 用户） tailscale funnel 1313 基于 WireGuard，需要 Tailscale 网络

8.3 实际操作：用 ngrok 暴露你的 Hugo 博客

这是最常见的做法，三步搞定：

  1. 下载 ngrok — https://ngrok.com/download （Windows 下载 exe 放到任意目录）
  2. 启动 Hugo — cd blog && hugo server
  3. 启动隧道 — 新开一个终端，ngrok http 1313

终端会显示一个公网 URL：

Forwarding   https://a1b2c3d4.ngrok.io → http://localhost:1313

把这个 URL 发给任何人，他们就能看到你的博客了。

🔐 ngrok 免费版的限制

免费版每次启动 URL 会变化（无法固定），每分钟限 40 个连接。这对"给朋友看看"绰绰有余，但不能用于正式服务。免费版还会在页面上显示 ngrok 的横幅广告。
