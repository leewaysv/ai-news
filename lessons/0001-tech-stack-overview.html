1. 三个层次

这个系统由三层组成，每层解决一个不同的问题。把它们想象成一家新闻编辑室：

层次 类比 技术 职责
Pipeline 记者 + 编辑 Python 脚本 搜集新闻、筛选、写稿
Blog 排版工 Hugo + Go 模板 把稿件排版成网页
Infrastructure 发行部 GitHub Actions + Cloudflare 定时发布 + 分发
💡 先看宏观再深入细节

如果你现在对各层具体怎么工作还不清楚，没关系。这节课的目标是让你在脑子里构建一张地图——以后深入每一层时，你都知道它在地图上的位置。

2. 数据流：从新闻源到你的浏览器

每天早上，一条生产线自动运转。这是完整的数据流：

┌─────────────────────────────────────────────────────┐ │ Pipeline（Python） │ │ │ │ RSS/API 🌐 → 预过滤 🔍 → LLM 摘要 🧠 → QA 校验 ✅ │ │ │ └──────────────────────┬──────────────────────────────┘ │ 生成 .md 文件 ▼ ┌─────────────────────────────────────────────────────┐ │ Blog（Hugo） │ │ │ │ .md 文件 → Hugo 构建 → HTML + CSS │ │ │ └──────────────────────┬──────────────────────────────┘ │ 触发部署 ▼ ┌─────────────────────────────────────────────────────┐ │ Infrastructure（CI/CD + Hosting） │ │ │ │ GitHub Actions → Hugo Build → Cloudflare Pages │ │ │ └─────────────────────────────────────────────────────┘ │ ▼ 你在浏览器看到的早报 🎉

这个流程完全自动。你不需要每天早上手动操作什么——管道在 GitHub 上定时触发，走完整个链条。

3. 每一层用什么技术？

3.1 Pipeline 层 — Python 脚本

Pipeline 是一个 Python 程序，你的项目里在 pipeline/ 目录下。它做这些事：

模块做什么关键文件
gather/ 从 RSS、API、网页抓取新闻 rss_collector.py
filter/ 判断文章是否和 AI 相关 classifier.py
summarize/ 调用 DeepSeek API 生成中文摘要 summarizer.py
qa/ 检查摘要质量 embedding_check.py
adapt/ 转成 Hugo 认识的 .md 格式 blog_adapter.py

3.2 Blog 层 — Hugo

Hugo 是一个静态站点生成器。它不是给你一个"后台管理界面"去一篇文章一篇文章地编辑——

它的工作方式是：

  1. 你在 content/ 目录放 .md 文件
  2. 你在 layouts/ 目录写 HTML 模板
  3. 你执行 hugo 命令
  4. Hugo 把 .md + 模板 合并成最终的 HTML 文件
  5. 这些 HTML 就是你的网站
🔑 关键概念：构建

"构建"就是执行 hugo --gc --minify 这个命令。它把 .md 和模板"编译"成 HTML。Pipeline 生成 .md 后，必须触发一次构建，新文章才能上线。这个过程完全自动——你不需要手动跑任何命令。

3.3 Infrastructure 层 — CI/CD

组件角色
GitHub Actions每天早上 06:00 自动运行 Pipeline
Git存储所有代码和文章 .md 文件
Cloudflare Pages托管网站，CDN 加速
Vercel (备选)另一个托管选项
